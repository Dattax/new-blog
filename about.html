About Us

A great many Helm charts have sailed into the ocean of the internet, and while a growing number of publicly available charts is great for the community, it can be hard to know where they all are. To help solve that, Helm created Helm Hub - a central catalog or index of the growing number of publicly available Helm chart repositories. Browsing the Helm Hub UI, you can see how easy it is to find lots of great Kubernetes-ready apps. Through your browser, you can search for the publicly available charts you need and even find charts created by organizations. With such a large and growing set of listings, wouldn’t it be great if your build automation tools could use Helm Hub as the single central location to pull Helm charts for your entire organization? But that’s not easy to do; you can’t access Helm Hub through the Helm CLI. To remedy, this the JFrog Community Team has created the Helm Center - a free central repository for Helm Charts.

Screenshot

What is the Helm Center?

HelmCenter is similar to Helm Hub in the way that it gives access to Helm charts, but it’s a true central repository instead of just a catalog of charts - and it contains additional features such as:

  • All helm charts are immutable
  • Provides robust metadata around every chart version, apiVersion, and appVersion.
  • You can point your helm client to it as a single source of truth.
  • You can browse chart’s dependencies which are docker images (all docker layers are shown per docker image) and subcharts.
  • And another very cool feature is a Security tab which shows if any docker image vulnerabilities.

To learn more about HelmCenter, read the blog posted by Helm co-founder Rimas M.
